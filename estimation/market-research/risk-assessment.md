# Évaluation des Risques : Spatial Flow (2026-2029)

*Date : Mars 2026*
*Données : Analyse interne Gemini (Simulation)*

## 1. Risques de Concurrence & Marché

| Risque | Probabilité (1-5) | Impact (1-5) | Stratégie de Mitigation |
| :--- | :---: | :---: | :--- |
| **Framer Motion ajoute le "Native AAA"** | 4 | 5 | Se différencier par les "Soul Constants" (synchronisation Figma-vers-Code unique). |
| **Saturation du marché des frameworks** | 3 | 3 | Cibler des niches verticales (GovTech, HealthTech) qui ont des obligations légales AAA strictes. |
| **IA (v0, Bolt) remplace le besoin de Framework** | 4 | 4 | Créer un connecteur Spatial Flow pour ces outils IA afin de devenir leur standard de sortie. |
| **Acquisition de Figma par un géant (ex: Apple)** | 2 | 3 | Maintenir l'indépendance technologique en restant compatible avec la spec W3C Design Tokens. |

## 2. Risques Technologiques

| Risque | Probabilité (1-5) | Impact (1-5) | Stratégie de Mitigation |
| :--- | :---: | :---: | :--- |
| **CSS Scroll-Driven Animations remplace le JS** | 4 | 3 | Intégrer nativement ces APIs CSS dans le framework pour alléger le runtime JS. |
| **WebGPU rend le moteur spring JS obsolète** | 2 | 2 | Étudier l'accélération WebGPU pour les animations complexes (Reflex Matrix). |
| **Changement majeur de React (ex: React 20)** | 2 | 4 | Rester "Framework Agnostic" au maximum dans le moteur de calcul des constantes. |

## 3. Risques Réglementaires & Légaux

| Risque | Probabilité (1-5) | Impact (1-5) | Stratégie de Mitigation |
| :--- | :---: | :---: | :--- |
| **Assouplissement des lois sur l'accessibilité** | 1 | 4 | Peu probable (la tendance est inverse). Continuer à promouvoir le AAA comme un avantage concurrentiel UX. |
| **Violation de brevet par un tiers (IP Theft)** | 3 | 5 | Suivre le plan de protection IP (enveloppe Soleau, enregistrement marque, open-core licensing). |
| **Nouvelles normes WCAG 3.0 contradictoires** | 2 | 3 | Participer aux groupes de travail W3C ou suivre de près les brouillons (Working Drafts). |

## 4. Risques Business (Opérationnels)

| Risque | Probabilité (1-5) | Impact (1-5) | Stratégie de Mitigation |
| :--- | :---: | :---: | :--- |
| **Manque de fonds (Burn rate)** | 4 | 4 | Stratégie Open Core pour attirer une communauté, puis Licensing Enterprise pour générer du cash-flow. |
| **Difficulté de recrutement d'experts WCAG** | 3 | 3 | Automatiser un maximum de règles d'accessibilité dans le framework pour réduire le besoin humain. |

## 5. Synthèse & Score de Risque Global

- **Score Moyen Probabilité** : 2,9 / 5
- **Score Moyen Impact** : 3,8 / 5
- **Verdict** : Le projet présente un profil de risque **Moyen-Élevé**, typique des innovations de rupture dans un marché saturé. La force de Spatial Flow réside dans sa **compliance réglementaire AAA native**, qui est son meilleur bouclier contre la concurrence purement esthétique.
