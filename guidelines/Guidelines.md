**DataVibe Design System Guidelines**

These guidelines serve as the single source of truth for the DataVibe codebase, mirroring the Figma structure.

## 1. Typography Architecture

We strictly separate **Branding** (Logo) from **Product** (UI) typography to ensure semantic integrity.

### Branding (Logo)
*   **Font**: `Datavibe` (Custom font, Bold only).
*   **Usage**: EXCLUSIVELY for the main application logo.
*   **Responsive Behavior**: Controlled via Figma Variables modes mapped to CSS classes.
    *   **Default**: 28px (Mob) -> 36px (Tab) -> 48px (Desk)
    *   **Splash**: 48px (Mob) -> 64px (Tab) -> 80px (Desk)
*   **Code Implementation**:
    *   Use the CSS classes `.text-logo-default` or `.text-logo-splash`.
    *   **Do not** write arbitrary Tailwind sizes (e.g., `text-3xl`) for the logo.
    *   **Do not** manually apply `font-bold` if the class handles it.

### Product (UI)
*   **Font**: `Manrope` (Google Font).
*   **Usage**: All standard UI elements (Headings, Body, Captions, Buttons).
*   **Implementation**: Use standard Tailwind classes (`text-sm`, `font-medium`, etc.) or semantic typography classes if defined.

## 2. Color System & Tokens

We use a semantic token system. **Never use hardcoded hex values or Primitive variables (e.g., `Color/Brand/600`) directly.**

### Logo Semantics
*   **Primary Text ("DATA")**: `text-logo-primary`
    *   *Light*: Neutral 900
    *   *Dark*: Neutral 0
*   **Accent Text ("VIBE")**: `text-logo-accent`
    *   *Light*: Brand 600
    *   *Dark*: Brand 400 (Adjusted for contrast on dark backgrounds)

### General Semantics
*   **Surfaces**: `bg-background`, `bg-card`, `bg-muted`.
*   **Text**: `text-foreground`, `text-muted-foreground`.
*   **Borders**: `border-border`, `border-input`.
*   **Actions**: `bg-primary`, `text-primary-foreground`.

## 3. Components

### `<Logo />`
*   **Location**: `src/app/components/branding/Logo.tsx`
*   **Rule**: Always import and use this component. Do not inline the SVG/Text markup elsewhere to ensure consistency.
*   **Structure**: It encapsulates the `.text-logo-default` class and semantic color classes automatically.

## 4. Figma Sync Protocol

The code reflects the Figma structure generated by our strict scripts:

1.  **Primitives** (`scripts/figma/1-primitives.js`): Defines raw hex values.
2.  **Semantics** (`scripts/figma/2-semantics.js`): Maps primitives to semantic tokens (e.g., Logo colors).
3.  **Typography Variables** (`scripts/figma/3a-typography-variables.js`): Defines responsive sizes.
4.  **Typography Styles** (`scripts/figma/3b-typography-styles.js`): Binds fonts and sizes to Text Styles.

**Workflow**: Any design change must originate in these scripts, be generated in Figma, and then synced to `src/styles/theme.css`.
